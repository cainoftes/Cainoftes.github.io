前言

&emsp;&emsp;接触的第一门编程语言是C。学习C的时候磕磕绊绊，有很多地方不够精通。后面又解除了一堆的语言，却总是摆脱不了C对我的影响。

&emsp;&emsp;程序有三种结构：顺序、选择、循环。有了这三种结构之后，理论上就可以表达所有的语义（对C来说，又提供了goto语句，所以可以不需要循环，但是这样会破坏它本身的结构化，结构化编程有它自己的优势。）。这就是为什么C的地位这么高的一部分原因——它刚好卡在了一个精妙的边界上，删减一部分之后，C会变成低级语言，增加一部分之后，C又会变得臃肿。

&emsp;&emsp;但是正是基于以上原因，后面学习的所有语言，我都当成了C来使用，不管是Java，Python，C#，还是C++。更确切来说，是把他们当成C + API来使用，直到前几天对比了十几份实现同一功能的Python程序，才发现我真的是在写C，满满的方言味。同时也第一次见识了Python的华丽和强大，我写的复杂逻辑，Python本身就可以用一条语句直接调用！

&emsp;&emsp;所以我来改正自己狭隘的思想来了，做一份索引，方便以后的查找。是为序！

内置序列类型

容器序列

序列中的每一个元素都是对象的引用。这是实现容器序列可以存放不同类型的数据的基础。

扁平序列

序列中的每一个元素都是真正的值，所以它里边只能存放同一类、基础类型的元素，更详细一点，扁平序列对应的是一段连续的空间（内存空间）。

可变序列MutableSequence

不可变序列Sequence

更详细地，可以说Mutable继承了Sequence。
